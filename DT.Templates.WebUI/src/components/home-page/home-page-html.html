What's included in this project

Welcome to your new application! Here is a short guide to help find yourself around.

  • MVC

    This project is built on top of a stripped down ASP.NET MVC 5 Web Application template

  • TypeScript

    TypeScript has been created throughout the application and is compiled by gulp at build time into AMD Modules. To enable Build-on-Save (file watching), you can run watch.bat which will watch for changes to your css and TS and will rebuild both when changes occur.

  • AMD & RequireJS

    AMD was chosen as the module format, since TypeScript can directly build to that format, and RequireJS was chosen as the module loader. RequireJS is configured via /src/require.config.js and can load text via the requirejs-text requirejs plugin (which was installed via bower).

  • TypeLite

    TypeLite has been included as a nuget package that will generate TypeScript definitions files for your C# classes and enums. As an example, it is generating a definition for an example class and enum to start with.
    Source File: /src/TypeLite.Net4.d.ts.
    Generated Definition File: /src/TypeLite.Net4.tt
    Generated TypeScript File: /src/Enums.ts

  • Knockout

    Knockout 3.2 has been included via Bower, along with Knockout: projections, postbox, punches and validation

  • Bootstrap

    Bootstrap 3.1.1 has been included via Bower and is bundled as part of the CSS bundle. The JavaScript has been included in /src/app/startup.ts

  • Misc JS Libraries

    jQuery, jQuery UI, Crossroads, and Hasher have been included via Bower

  • Package Management

    This project uses the following package managers:

    • NuGet

      NuGet manages the C# dependencies, including MVC, etc.

    • NPM

      NPM is used for all node dependencies and they are only used via Gulp at build time. NPM lists its dependencies in /package.json
      Note:You will need to have Node installed and available from the command line for npm to restore packages

    • Bower

      Bower is used for most / all, client side library dependencies. Bower is configured to store packages in /src/bower_modules and lists dependencies in /bower.json
      Note:You will need to have git installed and available from the command line for bower to restore packages

  • Gulp

    Gulp is used to do all of the non C# related builds, so TypeScript, LESS, Glyphs, RequireJS optimizer, minification, etc are all built through Gulp.
    Note: This project shares build configurations for both MSBuild and Gulp, these configurations are "Dev", "Test" and "Prod". For more information about what is included in the gulp builds, see the gulp configuration file in /gulpfile.js

  • Client Side Unit Testing

    This project uses a couple of tools for unit testing, a framework and a runner

    • Jasmine

      Jasmine is the unit testing framework used. Unit tests are written in TypeScript and are runnable via the browser by navigating to /src/test. To add new unit tests to the project, you should write your unit test in TypeScript, like the other unit tests, and include them in /src/test/all-tests.ts via an /// <amd-dependency> tag
      Note: scaffolding for this is coming soon

    • Karma

      Karma is a test runner that allows us to run unit tests in a number of different ways. By default, it runs them from the command line and outputs test results in a jUnit format, for use with a CI platform such as Jenkins. Karma has been configured to run all tests referenced from /src/test/all-tests.ts (just like jasmine)

  • LESS

    LESS is built as part of a Gulp build step. Currently, only /src/css/styles.less is the only LESS file being built / included. This means that you will need to @import your files in there to include it in the main css bundle.

  • Glyphs

    Glyph fonts are generated at build time via gulp. See the Glyphs page for a list of all glyphs. There is one font created per folder inside the /glyph/icon-sets folder. For example, there are 3 folders, icons, set2, set3, so there are 3 font / css files generated for those folders.
